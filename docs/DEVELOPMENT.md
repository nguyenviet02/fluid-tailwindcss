# Development Guide

## Install

```bash
pnpm install
```

The workspace links the homepage to the root package with `fluid-tailwindcss: workspace:*`.

## Root Package Workflow

```bash
pnpm test:run
pnpm lint
pnpm build
```

Scripts from `package.json`:

- `build`: builds ESM, CJS, and declarations with `tsup`.
- `dev`: watches package build with `tsup --watch`.
- `test`: runs Vitest in watch mode.
- `test:run`: runs Vitest once.
- `lint`: runs `tsc --noEmit`.
- `prepublishOnly`: builds before publish.

## Homepage Workflow

```bash
cd homepage
pnpm dev
pnpm build
pnpm lint
pnpm format
```

The homepage imports Tailwind CSS and loads this plugin in `homepage/src/index.css`:

```css
@import "tailwindcss";
@plugin "fluid-tailwindcss";
```

## Release-Sensitive Files

- `package.json`: version, package exports, peer dependencies, published files.
- `tsup.config.ts`: build entry points and output formats.
- `src/index.ts`: root public API.
- `src/tailwind-merge/index.ts`: subpath public API.
- `README.md`: user-facing install and usage docs.

## Generated Files

`dist/` is generated by `pnpm build`. Do not make source changes there.

## Adding a Utility

1. Add a mapping in `src/utilities.ts`.
2. Add special registration in `src/index.ts` only if a normal declaration is not enough.
3. Add class groups and conflicts in `src/tailwind-merge/index.ts`.
4. Add tests for generated CSS and merge behavior.
5. Update README/homepage examples if the utility is user-facing.

## Debugging Plugin Output

The plugin can emit debug comments when `debug` is enabled in options. Use this for development only; debug warnings are intentionally quieter by default to avoid noise from mixed-unit theme scales.
